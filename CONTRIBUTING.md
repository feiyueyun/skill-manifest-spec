# Contributing to Skill Manifest Specification

Thank you for your interest in contributing to the Skill Manifest specification!

## How to Contribute

### Reporting Issues

- Use GitHub Issues for bug reports and feature requests
- Search existing issues before creating a new one

### Proposing Changes

For changes to the specification:

1. Open a GitHub Discussion in the "Ideas" category
2. Describe the problem and proposed solution
3. If there's community interest, we'll create a formal RFC

### RFC Process

> Coming soon — we're setting up the formal RFC process.

### Code Contributions

For the parsing libraries and examples:

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-change`)
3. Make your changes
4. Run tests (`go test ./...`)
5. Submit a Pull Request

## Style Guide

- Go code follows `gofmt` and `golangci-lint`
- JSON Schema follows Draft 2020-12
- Documentation in English
